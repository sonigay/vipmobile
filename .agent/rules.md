# 응답 언어 설정 모듈

## 기본 규칙
1. **모든 응답은 반드시 한국어(Korean)**로 작성합니다.
2. 기술적인 용어는 영어를 병기할 수 있으나, 설명은 한국어로 합니다.
3. 코드 내 주석(Comments)은 가능한 한 한국어로 작성하여 가독성을 높입니다.
4. 단계별 설명이나 가이드 역시 친절한 한국어로 제공합니다.
5. 기존코드를 리팩토링 하는 과정에서는 반드시 index.js.backup.original 파일을 참고 해야합니다.
6. index.js.backup.original 파일에서 레거시 코드를 참고할때는 구글시트이름과 범위를 정확히 참고해야합니다.
7. index.js.backup.original 파일은 코드줄이 40000줄 이상의 무거운 파일입니다. 직접 접근을 피해야합니다.
8. 구글시트와 수파베이스 를 동시에 사용하는 하이브리드 코드를 생성할때는 구글시트를 사용할때는 구글시트api 의 성능을 생각해 리밋 제한을 고려해야하며 수파베이스코드는 상대적으로 리밋제한을 제거하여 빠른 반응속도를 낼수있도록 해야합니다 또한 서로 선택에따라 전역이 아닌 개별 동적으로 제어할수있도록 해야합니다.
9. index.js.backup.original 파일의 계산 로직은 정교하게 구현되어있으므로 반드시 참고해야합니다.
10. 어플종합관리모드 에서는 기존 레거시 코드를 비교분석하여 변경된 라우트파일과 변경된 컴포넌트파일을 비교 분석하여 아직 구현이 안되서 누락된 라우트 파일및 구글시트 만 있고 수파베이스 코드는 아직 구현이 안된경우가 많습니다. 이경우 수파베이스 가 같이 작동하는 하이브리드 코드를 생성하는데 사용자가 실시간 확인하여 수파베이스로 전환할수있도록 해야합니다.
11. UI에서는 모드별, 상위탭별, 하위탭별, 탭마다의 정해진 구글시트명 별로 구현이 되어있는지 안되어있는지를 한눈에 시각적으로 확인가능하도록 해야합니다.
12. 어플종합관리모드의 탭구조는 퀵서비스관리, 데이터베이스관리, 버그관리(신규탭) 으로 구조합니다.
13. 버그관리 탭은 모드별, 상위탭별, 하위탭별로 기본적인 버튼을 눌러 앱의 작동을 테스트 해서 상세한 에러코드를 한줄로 표현해 즉각적인 유지보수를 하는데 도움을 주어야합니다. 이때 IDE 가 한줄의 메시지로 바로 진단할수있도록 원인을 명확하게 표현해야합니다.
14. 버그관리에서도 에러가 발생되는 원인을 index.js.backup.original 파일을 참고하여 정확한 원인을 파악하고 수정해야합니다.
15. 코드작성 및 코드 수정후에는 반드시 린트 에러와 구문 에러를 확인하고 수정해야합니다.
16. 새로운 기능 추가하거나 코드 수정후에는 로컬에서 서버를 실행하고 브라우저탭을 이용해 정상작동에 대한 기능을 반드시 테스트 해야합니다.
17. 테스트할때 로그인 은 '01081128119'로 입력하고 이후에 비밀번호창이 뜨면 '7985456'을 입력하면 접속됩니다. 해당 아이디와 비밀번호는 총괄권한레벨의 'SS' 레벨권한이 있는 아이디 입니다.
18. 자동승인 터미널 커맨드는 auto-execution-demo.md 파일에서 확인합니다. 해당 파일에 있는 커맨드는 사용자의 승인없이 안티그래비티 IDE에서 직접 실행해도 됩니다.
19. API 를 새로 생성시에는 반드시 index.js 파일이 아니라 라우트 폴더에 라우트모듈로 생성해야합니다. 라우트모듈이 생성되면 index.js 파일에 라우트모듈을 추가해야합니다.
20. `task.md`, `implementation_plan.md`, `walkthrough.md` 등 모든 아티팩트 파일의 내용 및 요약 정보도 반드시 **한국어**로 작성합니다.


## 예외 사항
- 에러 로그 원문이나 시스템 메시지는 있는 그대로 표시합니다.
- 특정 기술 표준이나 프로토콜 명칭은 유지합니다.
