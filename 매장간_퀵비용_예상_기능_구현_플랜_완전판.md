  - [x] 전화번호: 하이픈 제거

  - [x] 여러 업체 정보를 한 행에 저장 (업체1~5)
  - [x] 각 업체마다 배차속도, 픽업속도, 도착속도 정보 저장 (빠름/중간/느림)
- [x] 입력값 검증:
  - [x] 비용: 0보다 크고 1,000,000 이하
  - [x] 전화번호: 숫자만 허용
  - [x] 업체명: 1~50자
- [x] **양방향 자동 반영 기능** (추가 요구사항)
  - [x] **조건**: 같은 퀵서비스 업체일 경우에만 양방향 저장
  - [x] **의도**: 퀵서비스 업체마다 비용이 다를 수 있으므로, 입력한 업체 정보를 양방향으로 제공해야 함

  - [x] 예: "넘버원모바일2호(시흥대야) ↔ 정직폰강서점(서울마곡)" + 업체A(010-1234-5678, 7000원, 빠름/중간/빠름) 저장 시
  - [x] "정직폰강서점(서울마곡) ↔ 넘버원모바일2호(시흥대야)" + 업체A(010-1234-5678, 7000원, 빠름/중간/빠름)도 자동 저장
  - [x] 프론트엔드에서 양방향 저장 API 호출 (Promise.all 사용)
  - [x] **구현 방식**: `saveDataReverse`에 `companies: companiesData`로 같은 업체 정보를 그대로 사용
  - [x] **결과**: 양방향 모두 같은 업체 정보(업체명, 전화번호, 비용, 속도)로 저장됨

- 말풍선에서도 빠르게 "다른 업체 등록"을 열 수 있도록 UX 개선 요청
